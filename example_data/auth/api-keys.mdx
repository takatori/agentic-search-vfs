# API keys

API キーは、バッチ処理やサーバー間連携のように人間の同意画面を伴わない処理で使います。ユーザー本人の権限を委任する場合は OAuth を優先してください。

## 作成

開発者ダッシュボードの **Settings -> API keys** からキーを作成します。キーには用途がわかる名前を付け、必要最小限のスコープだけを付与します。作成後に完全なキー文字列が表示されるのは一度だけです。

## 保管

API キーはソースコードや設定ファイルに直接書かず、シークレットマネージャーから実行時に注入します。モバイルアプリや配布済みバイナリに埋め込むと利用者側で抽出される可能性があります。

## ローテーションと失効

担当者の異動、退職、漏洩の疑い、ログへの誤出力があった場合は、古いキーを失効させて新しいキーへ切り替えます。失効は数秒で反映されますが、処理中のリクエストはキャッシュされた検証結果で完了する場合があります。

## 検索実験での使い方

`grep -ri "APIキー" /auth` のようなキーワード検索では、この文書を直接見つけられます。一方で、認証情報の露出や失効対応について自然文で探した場合にも、漏洩時の対処に近い文書として発見されることを確認できます。
