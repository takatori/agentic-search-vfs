# OAuth

OAuth は、ユーザーのパスワードをアプリケーションに渡さずに API へアクセスするための認可方式です。API 呼び出しでは OAuth フローで得た **access_token** を `Authorization: Bearer ...` ヘッダーに指定します。

## 認可コードフロー

外部アプリケーションは開発者コンソールで登録し、`client_id` と必要に応じて `client_secret` を受け取ります。公開クライアントでは PKCE を使い、`code_challenge` と `code_verifier` によって認可コードの横取りを防ぎます。

ユーザーは同意画面を確認したあと、登録済みの `redirect_uri` に戻されます。戻り値には短命の認可コードと `state` が含まれるため、呼び出し元は CSRF 対策として `state` を必ず検証してください。

## トークン交換

認可コードはトークンエンドポイントで **access_token** と交換します。長時間のセッションでは `refresh_token` を利用できますが、更新トークンはローテーションされる場合があります。常にサーバーから返された最新値を保存してください。

## 運用上の注意

`access_token` や `refresh_token` の完全な値をログへ出力してはいけません。ユーザー委任のアクセスには OAuth を使い、サーバー間連携のように利用者が存在しない処理では API キーの利用を検討します。

この文書は公開可能な認証ガイドです。実際のキー漏洩対応手順は `/handbook/security/api-key-incident.mdx` を参照してください。
