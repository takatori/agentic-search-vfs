# Audit log

監査ログは組織全体の重要操作を記録する内部文書です。参照できるのは内部グループの利用者だけです。

## 記録されるイベント

認証成功と失敗、ロール変更、API キー作成と失効、billing 設定の変更、データエクスポート要求を記録します。各イベントには安定した `event_id`, `timestamp`, `actor`, `resource` が含まれます。

## 検索

監査 API は期間、実行者、イベント種別、リソース ID によるフィルターをサポートします。任意フィールドの全文検索はパフォーマンス上の制約があるため、可能な限り構造化フィルターを使います。

## 保持と改ざん防止

保持期間中のイベントは追記専用です。訂正は元の `event_id` を参照する新しいイベントとして記録します。法的保全が設定されている場合、自動削除は保全解除まで停止されます。

## アクセス制御

全イベントの閲覧とエクスポートは内部管理者だけが実行できます。PUBLIC や BILLING だけのプロファイルでは、このファイルは `ls`, `cat`, `grep`, `semantic_search` のいずれにも現れません。
