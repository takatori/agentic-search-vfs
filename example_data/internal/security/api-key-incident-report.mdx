# APIキー漏洩調査メモ

この文書は内部の調査担当者向けです。公開ハンドブックには載せない検知条件、調査クエリ、封じ込め判断を記載します。

## 調査観点

漏洩が疑われるキーの `key_id` を使い、過去 30 日の呼び出し元 IP、User-Agent、失敗率、対象エンドポイントを確認します。通常利用と異なる国やクラウドプロバイダーからのアクセスがある場合は優先度を上げます。

## 封じ込め

キーを revoke したあと、関連する OAuth クライアント、Webhook signing secret、CI/CD の環境変数も点検します。侵害範囲が広い場合は、同じプロジェクトに属するキーをまとめてローテーションします。

## 権限確認用メモ

この文書は `/handbook/security/api-key-incident.mdx` と意味的に近い内容ですが、内部限定です。PUBLIC プロファイルでは、認証情報の漏洩対応を自然文で探しても、このパスが返らないことを確認してください。
